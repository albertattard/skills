Q: What information is required to create and edit a PoC in the MVP?
A: Use triage-context fields only: `name`, `lifecycle state`, `internal owner`, and `customer`; the system records `creation timestamp`.

Q: How should blocked and stale PoCs be surfaced in the portfolio triage view?
A: Show every PoC in one portfolio list, with blocked and stale indicators visible on affected rows.

Q: What must a manual journal entry contain in the MVP?
A: A manager-entered free-text note, with system-recorded timestamp, selected manager attribution, and entry type.

Q: How should PoC creation and field edits be attributed in the MVP?
A: Record `created by`, `last updated by`, and `last updated at` on the PoC record.

Q: What must a system journal entry for a state transition show?
A: Previous state, new state, timestamp, and selected manager attribution.

Q: How should `internal owner` and `customer` be represented in the MVP PoC record?
A: Plain text fields.

Q: How should a manager select themselves for attribution in the MVP?
A: Choose from a configured manager list.
